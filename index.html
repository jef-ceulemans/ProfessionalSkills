Hieronder ga ik jullie 4 projecten uitleggen van de voorbije 2 jaren, waaronder individuele projecten en groepsprojecten.

Requirements Analysis BPU Code Kraks

Voor dit project werden we random verdeeld in teams door onze RA-docent. We moesten hierbij een bestaand proces makkelijker maken voor onze klant.


Het probleem dat we moesten oplossen was:


  • Code Kraks is een bedrijf die coding lessen aanbiedt aan kinderen van 6 tot 18 jaar. Ze bieden verschillende lessenreeksen aan gegeven door een x aantal leerkrachten. Op dit moment worden alle inschrijvingen manueel uitgevoerd. Er zou een webapplicatie ontwikkeld moeten worden om de administratie van dit bedrijf te vereenvoudigen.

Hierdoor hebben we een analyse/prototypes gemaakt voor een online webapplicatie waardoor dit hele process makkelijker zal worden. Deze analyse zullen we in het 2de semester verder uitwerken naar een echte webapplicatie.


Mijn aandeel in dit project was nogal verspreid. In het begin heeft iedereen meegeholpen om het use case diagram te maken. Dan werd er verdeeld wie welke use case ging uitwerken. Onder het uitwerken hoorde erbij dat je de

  • Functionality
  • Precondition (indien aanwezig)
  • Normal flow
  • Alternatives (indien aanwezig)
    • typte in een document.

Dan moest je ook prototypes maken van uw eigen use cases in Axure.

Mijn prototypes waren:

  • Feedback geven
  • Feedback raadplegen
  • Feedback beheren

Project PHP Wielerclub WTC Roland

Dit project begon eigenlijk al in de examenperiode van januari. Om het kort even te schetsen, we hadden in de laatste week van de examenperiode een projectweek, deze was in team verband. Tijdens deze projectweek moesten we met een klant praten, namelijk iemand van de wielerclub WTC Roland. Deze persoon gaf ons extra informatie over een tijdrit die deze wielerclub jaarlijks organiseert. Ze willen hiervoor een webapplicatie waarop mensen zich kunnen inschrijven. Maar voor de werknemers van de wielerclub moeten er ook opties zijn om de wedstrijd te beheren. Hieronder valt bijvoorbeeld het opstellen van starttijden en het bijhouden van rondes. Je kunt wel een idee krijgen wat voor soort webapplicatie ze wilden.


Wat was de bedoeling tijdens deze projectweek:


  • We moesten, net zoals bij de opdracht van RA, een use case-diagram maken. Ook moesten we een SQL-diagram maken met alle elementen die we in ons use case-diagram hadden opgenomen. Daarnaast moesten we ook prototypes maken van hoe de webapplicatie eruit zou zien. Dit kwam dus zo goed als overeen met het project van RA. Maar om het project PHP te kunnen uitvoeren, hadden we dit wel nodig.

Mijn bijdrage aan dit project was vrij divers. Ik heb geholpen bij het maken van het use case-diagram en het SQL-diagram. Daarnaast heb ik zelf ook een aantal prototypes uitgewerkt, namelijk alle mailtemplates, het home scherm en het aanmeldscherm voor de ploegen.

Vervolg

Dit was dus het essentiële deel om het project op een deftige manier te maken. Maar waar het project vooral om draaide, was dat wij een applicatie moesten ontwikkelen in PHP Laravel. Echter, voordat we deze applicatie konden maken, moesten we een use case-diagram, een SQL-diagram en natuurlijk de prototypes hebben. Hiervoor hadden wij dus de projectweek om dit in orde te brengen.


Als team zijn we begonnen met het opzetten van onze database, waar we allemaal gezamenlijk aan hebben gewerkt. Vervolgens hebben we ons Scrum-bord opgesteld. We hebben toen bepaald welke pagina's van onze applicatie als eerste klaar moesten zijn en hebben deze taken toegewezen aan individuele teamleden, zodat iedereen aan een specifieke pagina kon werken.


We hebben de webpagina's van de applicatie gebouwd op basis van de eerder gemaakte prototypes. Echter, sommige pagina's hebben we gezamenlijk ontwikkeld, omdat sommigen meer kennis hadden van bepaalde functionaliteiten.



Mijn aandeel in dit groepsproject was wel duidelijk. In het begin heb ik mee geholpen aan de database en heb ik ook een huisstijl gemaakt voor WTC Roland. Maar ik heb ook specifieke pagina's zelf gemaakt:

  • De ploeg aanmelden pagina
  • De ronde aftikken pagina
  • De footer en de header op de pagina's

Data visualization dashboard in Qlik

Voor het vak data visualization moesten we individueel een dashboard maken in Qlik. We mochten niet kiezen over welk onderwerp we een dashboard moesten maken. De docenten hadden ons data gegeven over verkeersongevallen in België gedurende een bepaalde periode. Deze data omvatte verschillende aspecten zoals het soort slachtoffer, welke gemeente,...


De bedoeling was dat we een verhaal gingen vertellen in ons dashboard. Met dit dashboard zou iemand een krantenartikel van moeten kunnen maken. Dus het was de bedoeling dat we bepaalde inzichten in de data moesten laten zien die je niet los in de data zou kunnen zien.

Java project Zoo

Dit is een project dat we vorig jaar hebben moeten maken. Ik heb dit project gekozen om beetje uit te leggen omdat, Java wel een taal was die mij lag. Plus het project zelf ging zeer vlot. Dit was wel een individueel project, wat ook wel zijn voordelen heeft want je bent hier niet afhankelijk aan iemand anders. Hierdoor heb je automatisch meer vrijheid, wat ik dus wel leuk vind.


Het project dat we moesten maken was:


  • We zijn bezig met een webproject gerelateerd aan dierentuinen. Elke dierentuin heeft verschillende delen waarin verschillende soorten dieren zijn gevestigd. We noemen deze delen "Dierenwerelden". Elke Dierenwereld heeft een verantwoordelijke medewerker, die voor de dieren in dat deel van de dierentuin zorgt. We willen weten hoeveel bezoekers een dierentuin bezoeken en wanneer een bezoeker een van de dierentuinen voor de eerste keer bezoekt, krijgt hij een persoonlijke code toegewezen (samengesteld uit een deel van de naam van die dierentuin en een nummer). We willen ook weten in welke dieren een bezoeker geïnteresseerd is. We noemen deze lijst met dieren een "Verlanglijst".
